## 2. Anthropic Accidentally Leaks 512,000 Lines of Claude Code Source via npm
**Summary:** On March 31, Anthropic inadvertently published the complete source code of its Claude Code CLI tool to the public npm registry via a misconfigured debug source-map file (59.8 MB `.map` file in package version 2.1.88). The error is traced to an unpatched bug in the Bun JavaScript runtime — which Anthropic acquired in late 2025 — that generates source maps in production builds without warning. The ~512,000-line TypeScript codebase was mirrored across GitHub and analyzed by thousands of developers within hours, revealing unreleased architecture including feature flags and a background-agent mode code-named "KAIROS." Anthropic confirmed the incident, stating "no sensitive customer data or credentials were involved." Security researchers also flagged a concurrent, separate supply-chain attack on the `axios` npm package affecting users who updated Claude Code between 00:21–03:29 UTC on March 31; The Register separately reported that Claude Code quota exhaustion became "the top priority for the team" the same day.  
**Why it matters:** Anthropic — which markets itself as the safety-first AI company — had its entire coding agent exposed to competitors via a packaging error, revealing unreleased architecture and internal codenames to any developer who looked. The coincidence of a source code leak, a concurrent npm supply-chain attack, and an infrastructure capacity crisis in a single 24-hour window creates compounding reputational and security risk for Claude Code at its peak enterprise adoption moment.
